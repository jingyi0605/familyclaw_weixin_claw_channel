# 微信 Claw 通道插件

这是 FamilyClaw 的微信 Claw 通道插件。

它的目标很直接：把微信账号登录、消息轮询、消息发送和运行状态管理封装在插件边界里，让宿主只处理标准化后的通道数据，不去理解任何微信私有协议细节。

## 适合什么场景

- 想把 FamilyClaw 接到微信 Claw 通道
- 需要通过扫码登录方式接入账号
- 希望把消息拉取、回复发送和运行态落盘都限制在插件自己的目录里

## 当前能力

- 支持扫码登录和登录状态刷新
- 支持轮询拉取微信消息，并做入站标准化
- 支持发送文本以及基础附件
- 支持插件私有 SQLite 运行状态持久化

## 当前媒体边界

- 出站支持宿主通用 `delivery.attachments`
- 支持的附件类型：`image`、`video`、`file`
- `audio` 出站按文件附件链路处理，不额外增加宿主私有字段
- 入站媒体通过标准附件引用回传给宿主

## 使用前需要知道的事

- 当前主链路是 polling，不是 webhook
- 运行时二维码、日志和本地状态会写入插件私有 `.runtime/` 目录，这些文件不应该提交到 Git
- 账号运行模式默认按稳定模式收口，调试模式只建议在排障时使用

## 已知限制

- 当前不支持群聊能力
- 当前不支持线程化会话能力
- 不把任何微信 Claw 私有实现反向塞回宿主核心

## 兼容性

- 插件版本：`0.1.0`
- 最低宿主版本：`0.1.0`
